About

Juggernaut - это библиотека (с претензией на фреймворк), которая расширяет возможности BitrixFramework, а также облегчает жизнь разработчику.

Можно использовать компоненты по отдельности в любых проектах, либо использовать в совокупности для разработки под Битрикс (juggernaut.bitrix).

Основная задача данного проекта: сделать приятной, удобной и простой разработку под Битрикс.

Get started

Установить модуль для Битрикс можно с помощью Compser: 

composer require irpsv/juggernaut.bitrix

В последствии, необходимо подключить файл autoload.php компосера, тогда можно будет напрямую обращаться к любым классам, без непосредственного подключения модуля с помощью Битрикс.

Если говорить про использование компонентов вне Битрикс, то подключение производиться также через composer.

Components

Каждый компонент отвечает за одну логическую составляющую веб-приложения.

Ниже представленны все компоненты, которые реализованы (или будут) в библиотеке Juggernaut.

Для каждого компонента представлен краткий пример и описание. Более подробный пример и документацию о каждом, можно узнать в соответствующих репозиториях.

Bitrix

Данный компонент, это сборка всех компонентов библиотеки и адаптациях их под Битрикс.

Собственно для разработки под Битрикс, достаточно подключить данный компонент, все остальные подтянутся автоматически.

ActiveRecord

Active Record (AR) - это объект, который является "обёрткой" одной строки из БД или представления, включает в себя доступ к БД и логику обращения с данными.

Паттерн древний, как сама жизнь, так что лишних объяснений думаю не нужно.

Однако, ряд функций вынесены в ServiceLayer, поэтому в реализации данного паттерна, выполняется ряд оговорок:

  • AR-объект один в этом мире - никаких данных о других таблицах, в реализации нет;
  • AR не содержит бизнес-логики - все логика и точки взаимодействия содержатся в ServiceLayer;
  • AR не содержит транзакций - по сути AR они и не нужны, т.к. все операции выполняются за одни запрос к базе.

Validator

Валидатор - это объект, который проверяет соответствие некого значения, некому условию.

В данном компоненте уже реализованы основные валидаторы, которые требуются в большинстве случаев:

  • RequireValidator - проверяет установленно ли значение
  • IntegerValidator - проверка на целочисленность
  • NumberValidator - проверка на число
  • LengthValidator - проверка соотвествия количества символов
  • RangeValidator - проверка на нахождение числа в указанном диапазоне
  • EmailValidator - проверка строки на соответствие формату электронной почты
  • UrlValidator - проверка строки на соответствие формату URL адреса
  • RegexpValidator - проверка строки на соответствие регулярному выржанию (большинство валидаторов реализованы на основе данного)

ServiceLayer

Service Layer (SL) определяет для приложения границу и набор допустимых операций с точки зрения взаимодействующих с ним клиентских, инкапсулирует бизнес-логику приложения, управляя транзакциями и управляя ответами в реализации этих операций.

Вся бизнес-логика и все взаимодействие между ActiveRecord содержится именно в реализации данных элементов приложения.
